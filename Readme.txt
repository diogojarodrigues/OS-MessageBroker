Este zip contem 4 pastas (assets, relatorios, src e tests)
 - A pasta assets contêm os ficheiros auxiliares disponibilazados pelos docentes da cadeira
 - A pasta relatórios contêm os relatórios em formato html aos testes fornecidos
 - A pasta src é composta por duas subpastas. Uma pasta server e outra pasta user
 - A pasta tests contem os testes fornecidos pelos professores

Para correr a aplicação basta, compilar os executáveis usando o comando "make" nesta diretoria,
de seguida basta correr o servidor usando o comando "./AS",
e por fim correr o cliente usando o comando "./user".

Observações:
 - Os dados dos users e auctions são guardas na pasta src/server/data, em relação à diretoria atual
 - As imagens após o comando "show asset", no lado do user, são guardas na pasta src/user/imagens
 - Todos os path são tratodos de forma dinâmica em relação à diretoria principal, portanto se as pastas
server e user forem mudadas de localização estes path terão de ser alterados
